Detailed Description

This class contains the operations that can be performed by the Mobile Client. Each operation is implemented as a static method.

There are three types of operations provided:

  • Audio recognition operations
  • Fingerprint generation operations
  • Search operations

To invoke an operation call the associated method. While an operation is running progress notifications can be received by the application. When an operation completes, results are delivered via a Result Ready Object. Operations can be canceled.

For more information, see the Mobile Client Implementation Guide.

static void com.gracenote.mmid.MobileSDK.GNOperations.fingerprintMIDFileFromFile ( GNFingerprintResultReady  resultReady,
GNConfig  config,
String  filename 
)
static

Generates a fingerprint for an audio file. The generated fingerprint can be used to search Gracenote's database for matches using Gracenote's MID-File product.

To process the audio file, Mobile Client requires write access to the directory where the audio file is stored. The storage media that contains the audio file must also have memory available for Mobile Client to use. The required space is estimated at 4Mb for fingerprinting a single file and up to 20Mb for fingerprinting multiple files concurrently.

Parameters
resultReadyResult Ready Object instance
configOperation configuration
