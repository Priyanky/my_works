Detailed Description

The GNConfig module implements a configuration object required when invoking an operation via the GNOperations module. You invoke GNConfig.init() and pass a Client Id String and an Android Context to create a GNConfig object.

Your Client Id is provided by your Gracenote Professional Services representative.

In a typical application the default configuration object can be used. If required, you can change the application-wide configuration properties by invoking the GNConfig.setProperty() method on the configuration object instance returned by GNConfig.init().

To create a custom set of configuration options that apply to a specific invocation of an operation, create a copy of the GNConfig object instance using its clone() interface. Then customize the properties on the created object instance and provide it when invoking operations that require the customized property settings.

Context  inContext 
)
static

Creates a configuration object that is used when invoking a Mobile Client operation via the GNOperations module.

Configuration objects must be created prior to invoking any Mobile Client operations. Create a configuration object by invoking this method. Be sure to store the returned object.

You must provide a Client Id and an Android Context. Your Client Id is provided by your Gracenote Professional Services representative.

The provided Context can be any Android Context; ensure it is valid for the life of the application. For example, an Activity Context may not be valid for the duration of the application; once the Activity object is destroyed, Mobile Client will generate an exception when it next attempts to access the Context object. Gracenote recommends providing the Application Context, which can be obtained from an Activity object. See the Sample Application for a complete example of creating and using a configuration object.

Returns
Configuration object instance initialized with the default property values.
